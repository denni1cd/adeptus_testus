## Behavioral Contract Implemented

The simulator now supports named deterministic scenario construction while preserving existing default behavior. The `default` named scenario uses the same 08:00 Dormitory start, baseline needs, and normal-day activity schedule as the previous no-argument `default_facility()` behavior. A deterministic `maintenance_day` scenario starts Sarah at 08:00 in Dormitory with a fixed maintenance schedule that moves through validated graph rooms and can run through the existing `advance_until()` flow.

Unknown scenario names fail predictably through `UnknownScenarioError` instead of silently falling back to a different scenario.
